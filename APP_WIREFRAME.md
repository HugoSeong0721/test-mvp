# APP Wireframe (MVP)

## Screen A. 환자 사전 문진 입력

### 목적
- 환자가 세션 전 핵심 상태를 빠르게 입력.

### 레이아웃
1. 상단 헤더
   - 클리닉명
   - "세션 전 체크인"
   - 진행률 바 (예: 3/10)
2. 본문
   - 카테고리 카드(한 번에 1개 또는 아코디언)
   - 질문 텍스트(환자 친화 문구)
   - 답변 입력창(단문/중문)
3. 강조 버튼 영역
   - [이게 메인 통증이에요]
   - [침술사가 기억해줬으면 해요]
4. 하단 CTA
   - [임시 저장]
   - [제출하기]

### 상태/예외
- 미입력 시 카테고리 "미완료" 표시.
- 제출 전 확인 모달:
  - "제출 후에도 수정할 수 있어요(세션 전까지)."

---

## Screen B. 침술사 대시보드 (환자 리스트)

### 목적
- 오늘 환자 중 누구를 먼저 봐야 할지 빠르게 판단.

### 레이아웃
1. 상단
   - 날짜
   - 필터(오늘/내일, 제출완료/미제출)
2. 환자 카드 리스트
   - 환자명 / 예약시간
   - 문진 제출 상태
   - AI 1줄 요약 미리보기
   - 강조 아이콘(메인 통증/기억 요청)
3. 우선 확인 섹션
   - 미응답 카테고리 많은 환자
   - 강조 항목 많은 환자

### 상태/예외
- 미제출 환자: "문진 미제출" 배지.
- 요약 실패 시: "요약 준비 중" + 새로고침.

---

## Screen C. 침술사 환자 상세 (세션 전 브리핑)

### 목적
- 30~60초 내 핵심 파악 후 세션 시작.

### 레이아웃
1. 상단 요약 박스
   - AI 3줄 요약
   - 마지막 업데이트 시간
2. 강조 항목 박스
   - 환자 표시 문장 그대로 노출
3. 카테고리 섹션 (TCM 10)
   - 각 카테고리별 답변
   - 미응답 카테고리 하이라이트
4. 세션 준비 메모
   - 침술사 개인 메모(로컬/클리닉 내부용)

### 상태/예외
- 데이터 없음: "환자 입력 대기 중"
- 미응답 카테고리: "세션에서 직접 확인 필요" 라벨

## 화면 전환 플로우
1. 환자: 문진 입력(Screen A) -> 제출
2. 시스템: 요약 생성
3. 침술사: 대시보드(Screen B) -> 환자 상세(Screen C)

- 금지: "진단", "확정", "치료 처방"

## 디자인 원칙 (MVP)
- 고령 사용자 고려: 큰 폰트, 큰 터치 영역, 단순한 문장.
- 한 화면 한 목적.
- 제출/강조 버튼은 항상 화면 하단에서 보이게 유지.
