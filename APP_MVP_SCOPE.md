# APP MVP Scope (v1)

## 1) 제품 한 줄 정의
- 세션 전 환자 입력을 받아 AI가 3줄 요약을 만들고, 침술사가 세션 전에 핵심을 빠르게 파악하도록 돕는 앱.

## 2) MVP 목표
- 환자가 세션 전 문진을 완료한다.
- 침술사가 세션 전 30~60초 내 환자 상태를 파악한다.
- "미응답 항목"을 확인해 세션에서 추가 질문 포인트를 잡는다.

## 3) MVP 사용자
- 1차 사용자: NJ/NY 한인 침술사 + 해당 클리닉 환자.
- 2차 확장 대상: 일반 침술 클리닉.

## 4) MVP 핵심 플로우 (고정)
1. 환자가 세션 전 질문지(TCM 10 카테고리 기반)를 입력한다.
2. AI가 답변을 카테고리별로 정리하고 3줄 요약을 만든다.
3. 침술사 대시보드에서 다음을 확인한다.
   - 카테고리별 요약
   - 환자 강조 항목
   - 미응답 카테고리

## 5) MVP 기능 포함 (In Scope)
- 환자 앱/웹 화면
  - TCM 10 카테고리 기반 문진 입력
  - "이게 메인 통증이에요" 강조 버튼
  - "침술사가 기억해줬으면 해요" 강조 버튼
- AI 처리
  - 카테고리 분류 정리
  - 3줄 핵심 요약
  - 강조 항목 추출
- 침술사 대시보드
  - 환자별 요약 카드
  - 카테고리별 답변 유무 표시
  - 미응답 카테고리 하이라이트
- 기본 운영
  - 로그인(간단한 계정 체계)
  - 환자-클리닉 연결(코드 또는 링크)

## 6) MVP 제외 (Out of Scope)
- AI 진단/질병 추정 문구 노출
- 인앱 쇼핑
- 교육 콘텐츠 탭
- 음성 입력
- 푸시 알림 고도화(Hook Notification 실험)
- 세션 간 라이프로그(수면/식단/커피/걷기 등)
- 결제/프리미엄 과금
- 외부 EMR/API 정식 연동

## 7) 성공 지표 (MVP)
- 사전 문진 완료율: 예약 환자 중 제출 비율.
- 대시보드 열람률: 세션 전 침술사 열람 비율.
- 사전 파악 시간 단축: 초진/재진 기준 체감 시간.

## 8) 규정/리스크 가드레일
- 의료 조언/진단처럼 보이는 표현 금지.
- MVP 문구는 "요약/정리" 중심으로 제한.
- 개인정보 최소 수집 원칙 적용.
- HIPAA 준비를 전제로 데이터 저장/접근 로그 구조 설계.

## 9) MVP 완료 정의 (Definition of Done)
- 환자 문진 제출부터 침술사 대시보드 확인까지 실제 동작.
- 실제 테스트 계정 5명 이상으로 end-to-end 검증 완료.
- 핵심 플로우 오류 없이 1주 이상 운영 가능.

## 10) 바로 실행할 개발 순서
1. 데이터 구조 확정 (질문, 답변, 요약, 강조, 미응답)
2. 환자 입력 화면 구현
3. 요약 생성 로직 연결
4. 침술사 대시보드 구현
5. 내부 테스트 및 문구/UI 정리
